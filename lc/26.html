26. Remove Duplicates from Sorted Array
                       
                    
class Solution(object):
    def removeDuplicates(self, nums):
        """
        :type nums: List[int]
        :rtype: int
        """
        
        seen = []
        p1_index = 0
        dupFound = False
        
        seen.append(nums[0])
        
        i = 1
        # iterate through array
        while (i < len(nums)):                
            # skip over duplicates
            while (nums[i] in seen):
                dupFound = True
                i += 1
                if (i >= len(nums)):
                    break
            if (i >= len(nums)):
                break
            if (dupFound):
                # move p1 forward
                p1_index += 1
                # swap unique with duplicate
                temp = nums[p1_index]
                nums[p1_index] = nums[i]
                nums[i] = temp
                if (nums[p1_index] not in seen):
                    seen.append(nums[p1_index])
            else: # no dupes, move both pointers forward
                seen.append(nums[i])
                i += 1
                p1_index += 1
            dupFound = False  # reset dupe marker
        return len(seen)

Given an integer array nums sorted in non-decreasing order, remove the duplicates in-place such that each unique element appears only once. The relative order of the elements should be kept the same. Then return the number of unique elements in nums.

Consider the number of unique elements of nums to be k, to get accepted, you need to do the following things:

  • Change the array nums such that the first k elements of nums contain the unique elements in the order they were present in nums initially. The remaining elements of nums are not important as well as the size of nums.

  • Return k.

My solution: My initial approach to this problem was to take every duplicate value in the array and shift it all the way rightward, leaving behind only the unique values in the front of the array. This worked for 360/362 test cases, but failed on the last two, which involve very large arrays with many duplicates, causing the algorithm to time out. So while such an algorithm does work for smaller arrays, it breaks down when we are dealing with arrays at scale.

So, how to solve this for all array sizes? The hints are fairly useful, namely skipping over duplicates rather than shifting them about. What I ended up doing was keeping a list of values that have already been seen, and adding each unique element in nums to this list when encountered.

The iterations, as in problem 27, involve maintaining multiple pointers. Start by creating two pointers - one to keep track of the last position near the beginning of the array - this will be where you swap unique values to. The second pointer is the one that iterates through the array from front to back.

Whenever you encounter a duplicate (found in the seen array), simply continue iterating past it with your second pointer until a unique value is reached. Then, increment your first pointer by 1 and swap the values of the two pointers. This has the effect of swapping out duplicates for new, unique values found later in the array.

Keep iterating until the end of the array, moving the second pointer past duplicates, incrementing the first pointer when a unique value is found, and swapping. If there are no duplicates found in one interation, simply increment both pointers by 1 and continue with the above process. I used a boolean variable to determine when duplicates were present and when they weren't, resetting this after each iteration.

Since you will need to return a count of the number of unique elements, you can simply look to your seen array, since this will contain all unique values seen at least once. Make sure you don't add duplicates to this array while iterating.
