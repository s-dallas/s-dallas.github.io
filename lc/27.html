27. Remove Element
                    
class Solution(object):
    def removeElement(self, nums, val):
        """
        :type nums: List[int]
        :type val: int
        :rtype: int
        """
        i = len(nums) - 1
        while (i >= 0):
            if (nums[i] != val):
                i -= 1
                continue
            else:
                j = i
                k = i + 1
                while (k <= len(nums) - 1 and nums[k] != val):
                    nums[j] = nums[k]
                    nums[k] = val
                    j += 1
                    k += 1
                i -= 1
        count = 0
        for n in nums:
            if (n != val):
                count += 1
        return count

Given an integer array nums and an integer val, remove all occurrences of val in nums in-place. The order of the elements may be changed. Then return the number of elements in nums which are not equal to val.

Consider the number of elements in nums which are not equal to val be k, to get accepted, you need to do the following things:

Change the array nums such that the first k elements of nums contain the elements which are not equal to val. The remaining elements of nums are not important as well as the size of nums. Return k.

My solution: This problem can be trivially easy by simply using the built-in pop() or remove() methods in Python. However, this is not what I believe was intended with the problem, so my solution does not use these.

Since you only need to retain the elements in nums that aren't the specified val, you can begin at the end of the list, iterate through to the front, and each time you encounter val, continually swap it with its right-hand neighbor to get it as far to the right of the list as possible.

To do so, you need to keep track of 3 variables: The position of your pointer as you iterate right to left, and the positions of the two pointers which indicate the values to be swapped as you shift around the elements. I have called these i, j, and k, respectively.

The idea is that you first move left through the list continuously until you come upon a position that contains val. Keep this position stored in i, then define j to initially be the same as i and k to be one position to the right of that (j+1). Swap j and k until you either reach the right-hand side of the list or you reach a position that already contains val from an earlier swap. This is the inner while loop.

After getting this element out of the way, you can then decrement i in the outer loop and continue moving left toward the front of the array. Once you reach the start of the array (i = 0), you will have ensured that all instances of val have been moved rightward in the array, leaving only non-val values remaining in the first half of the array.

At this point, it's only a matter of counting those non-val elements by iterating back through the list from front to back until val is encountered and keeping a running counter. This counter is what you will be returning. (I have unfortunately used k as one of my pointer variables instead of the value to be returned as described in the exercise prompt, so just recall that my k is not what it signifies in the prompt. I simply used a different int variable for my final count.)
