941. Valid Mountain Array

Given an array of integers arr, return true if and only if it is a valid mountain array.

>Recall that arr is a mountain array if and only if:

  • arr.length >= 3

- There exists some i with 0 < i < arr.length - 1 such that:

  • arr[0] < arr[1] < ... < arr[i - 1] < arr[i]

  • arr[i] > arr[i + 1] > ... > arr[arr.length - 1]

My solution: You will want to split this problem into two parts. First, you will need to check the increasing portion. Iterate through the array, checking whether each subsequent element is larger than the previous. If so, keep iterating. Once that is no longer the case, break out of the loop.

There are 4 cases at this point: You have reached the end of the array, which means the array consisted of increasing terms only, none decreasing, so you can immediately return False. Case 2 is that you moved laterally at the end of the loop - that is, there were duplicate adjacent values. This is not strictly increasing, so you can return False. Case 3 is that you didn't move anywhere, meaning the array does not begin with increasing terms, which again leads to a False outcome. Lastly, you are somewhere in the array after a series of increasing values.

In this last case, you must then check the decreasing part of the mountain. This is essentially the same as as the first iteration, only now you are checking whether each subsequent term is smaller than the previous. At the end of this loop, there is only one suitable outcome, namely that you are just past the end of the array. Check the index to make sure this is the case, and return True if so, false otherwise. If you aren't at the end, it means the array incnreases again at some point, so it's not a mountain.
