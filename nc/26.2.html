26.2 - Remove Duplicates from Sorted Array (Efficient)

Given an integer array nums sorted in non-decreasing order, remove the duplicates in-place such that each unique element appears only once. The relative order of the elements should be kept the same. Then return the number of unique elements in nums.

Consider the number of unique elements of nums to be k, to get accepted, you need to do the following things:

  • Change the array nums such that the first k elements of nums contain the unique elements in the order they were present in nums initially. The remaining elements of nums are not important as well as the size of nums.

  • Return k.

My solution: This is an alternate solution to problem 26 which does not use additional memory and which runs in linear time. The goal is to remove all duplicate entries in an array in-place without using nested loops (i.e., O(N) runtime).

Rather than use a separate array to store seen values as we did in the previous solution, we're just going to store the last seen number in an int variable. Once again, we will be using two pointers, one to iterate through the array, the other to keep track of where to swap values.

We can start with edge cases, as they are the easiest to consider. Arrays of length 1 will always return size 1. Arrays of length 2 can return either length 1 or 2 depending on if it is 2 duplicate or unique numbers.

Then we go into our main iteration loop. Both pointers start at the second item in the array. The pointer i advances so long as it sees repeat values, stopping when it reaches a unique value. The unique value swaps places with the pointer s, and then both s and i are incremented by 1, as well as the size variable (which begins at 1 to take into account the first element of the array.

With that basic idea in mind, there are really only 2 cases to consider in the loop. Either the array is filled with unique values, in which case i simply advances to the end of the array, or there is a duplicate that i stops at. In the former case, we simply return the current size because there are no longer any elements to check. In the latter, we swap the value in i into the position of s, store the new duplicate value in last, increment both pointers, increment the size by 1, and continue into the next iteration of the loop.

Since we are only making one pass through the array (swapping when necessary and incrementing the size counter), this runs in linear time. Since we are not using any other data structures, its memory complexity is O(1), as required by these new, alterative requirements for the problem.
